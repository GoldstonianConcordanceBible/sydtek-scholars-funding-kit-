# Incident Response (Security/Privacy)

## Purpose
Respond consistently to security/privacy incidents.

## Examples
- lost device containing restricted data
- unauthorized account access
- accidental sharing of student info

## Steps
1) Contain: revoke access, secure accounts, preserve evidence
2) Assess: what data, how many affected, scope
3) Notify internal roles: Security Lead + Program Director + Highest Authority
4) Remediate: reset credentials, update controls, retrain staff
5) Document: restricted incident report stored privately
6) Improve: update policies and training to prevent recurrence
